# Topic Guides

Start here when searching by programming problem. These pages synthesize tips across multiple videos and link back to the source notes.

- [AI Engineering Guardrails](../topics/ai-engineering.md) - Use strict project defaults so AI-generated code is easier to review, test, and trust.
- [Clean Laravel Code](../topics/clean-code.md) - Keep code readable by giving each layer one job: requests validate, controllers orchestrate, actions perform behavior, and tests describe outcomes.
- [Database Integrity](../topics/database-integrity.md) - Protect application state with explicit constraints, transactions, and schema changes that match production behavior.
- [Dependencies And Maintenance](../topics/dependencies-and-maintenance.md) - Keep collaborators, packages, and upgrade work explicit so the codebase stays understandable and safe to evolve.
- [Developer Tooling](../topics/tooling.md) - Make quality tools easy to run, easy to automate, and boring enough that the team actually uses them.
- [Domain Modeling In PHP](../topics/domain-modeling.md) - Use types, enums, value objects, and clear object relationships to make business rules explicit.
- [Event Sourcing](../topics/event-sourcing.md) - Use event streams when history, auditability, temporal reporting, or process reconstruction is central to the domain.
- [Frontend Integration](../topics/frontend.md) - Keep frontend choices type-safe, framework-aware, and connected to the backend contracts they depend on.
- [Laravel Actions](../topics/laravel-actions.md) - Move reusable application behavior into action classes so controllers, jobs, commands, and other actions can share it.
- [PHPStan And Type Safety](../topics/phpstan-type-safety.md) - Use static analysis and native types to catch defects before runtime and to give AI tools a stricter contract.
- [Request Data Boundaries](../topics/request-data-boundaries.md) - Validate raw input at the edge, then pass typed, trusted data into application behavior.
- [Testing Strategy](../topics/testing.md) - Use fast, focused tests plus coverage and quality gates so code changes are safe to accept.
