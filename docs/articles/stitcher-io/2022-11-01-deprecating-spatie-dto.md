## Why This Helps Programming
A useful reminder that packages should become obsolete when the language and ecosystem grow better primitives for the same job.

## Guidelines
### Do not keep a package only because it was once essential
When PHP native types, static analysis, or better mapping libraries cover the original need, retire or replace the older abstraction instead of stretching it into a different tool.

### Choose DTO tooling for its current job
Runtime type checking, request mapping, serialization, and validation are related but different problems. Pick tools that match the job instead of expecting one DTO package to own all of them.
